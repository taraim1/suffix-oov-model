## 💡 핵심 로직 (Core Logic)

- 학습 데이터에 없는 미등록어 판별 문제 해결을 위해, 접미 형태소열의 공기(Co-occurrence) 정보를 활용하여 유사한 문맥을 공유하는 지표 형태소열을 찾고, 이를 통해 웹에서의 사용 빈도를 추정하여 미등록어 여부를 확인하는 확률 모델입니다.

1.  **후보 생성:** 어절(예: `유로존에서는`)을 쪼개어 후보(`유로존` + `에서는`)를 생성합니다.
2.  **지표 형태소($S^B$) 탐색:** `에서는`과 유사한 패턴을 가진 다른 접미사들(`에서`, `에는` 등)을 찾습니다.
3.  **확률 계산 (Score):**
    -   **Local Score:** 말뭉치 내에서 해당 접미사가 명사 뒤에 얼마나 자주 붙는지 통계
    -   **Web Score:** 네이버 뉴스 검색을 통해 후보 단어의 실제 사용 빈도 추적
4.  **판별:** 최종 점수가 임계값을 넘으면 미등록어(신조어)로 등록합니다.

### 2. 데이터 준비 (Data Preparation)
본 리포지토리는 언어정보나눔터의 규정으로 인해 **말뭉치 원본 데이터를 포함하지 않습니다.**

-   **테스트 실행:** `data/` 폴더 내에 포함된 `corpus.json` 샘플을 사용하여 모델의 동작을 검증할 수 있습니다.
-   **전체 실행:** 언어정보나눔터 모두의 말뭉치 중 `형태 분석 말뭉치`(Json 형식)를 준비하여 파일 이름을 `corpus.json`으로 변경한 뒤 샘플 파일을 대체해 주세요.

```text
project_root/
├── data/
│   ├── corpus.json  # (기본 제공) 테스트용 샘플 데이터
│   └── (corpus.json)  # (사용자 준비) 전체 학습 데이터로 대체
```

### 3. 전처리 모듈 실행 (Preprocessing)
#### 현재 이곳까지 구현되어 있습니다.

- 1. 프로젝트 클론 단계에서 열었던 cmd창에 아래 명령어를 한 줄씩 붙여넣고 실행해 주세요.

```bash
cd suffix-oov-model
python preprocess.py

```

실행이 완료되면 `data` 폴더 안에 `model_data.json` 파일이 생성됩니다. 이 파일은 다음 정보들을 포함합니다.
*   `S_total_count`: 접미 형태소열 전체 등장 빈도
*   `S_noun_count`: 명사 후행 접미 형태소열 등장 빈도
*   `co_occurrence_frequencies`: 지표 형태소열($S^B$) 추출을 위한 상위 $K$개 공기 빈도 정보

## 🚀 트러블슈팅 로그

> **개발 과정에서 마주친 기술적 난관을 기록했습니다.**

### 삼중 for문 + 교집합 연산의 시간 복잡도 문제
- 공기 빈도를 구하는 과정에서 3중 for 문이 사용되었고, 그 안에 집합 교집합 연산이 들어가 시간 복잡도가 높아졌습니다.
- 해결책으로 카운터를 사용한 알고리즘을 적용했습니다.
- 기존 구현은 (S, SB) 쌍마다 교집합을 수행하는 반면 개선된 구현은 시간복잡도가 O(1)인 카운터 연산을 사용했기에, 해당 비용을 제거하였습니다.
- 다만 기존의 15 문장 분량의 샘플 말뭉치를 단순히 100000배로 늘린 케이스에선 거의 차이가 없거나 오히려 기존의 교집합 방식이 빠른 현상이 나타났습니다.
- 이 현상의 원인은, 단순히 자료를 복사 붙여넣기 하는 것으로는 명사와 접미 형태소열의 종류가 늘어나지 않았기 때문인 것으로 보입니다.
- 그러나 만약 샘플 말뭉치가 아닌 실제 말뭉치를 사용한다면 그 차이가 드러날 것이라고 예상됩니다.
